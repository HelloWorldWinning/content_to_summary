**引言**
本视频详细介绍了 CodeX，一款当前热度高、对中国用户友好且功能强大的 AI 编程工具。相比之下，Cloud Code 近期表现平平。视频内容涵盖 CodeX 的四种运行环境、一系列进阶功能、免费模型接入方法，并通过实际编程案例对比了其与 Cloud Code 的编程效果。

**一、核心优势**
*   **便宜大碗**：CodeX 的 Plus 会员额度充足，个人使用很少触发速率上限。
*   **对中国用户友好**：相较于其他工具，CodeX 极少出现封号等问题。
*   **功能强大且灵活**：提供多种运行环境，支持自定义和扩展。

**二、CodeX 四种运行环境**

1.  **CRI (命令行环境)**
    *   **安装与启动**：首先安装 Node.js，然后通过 GitHub 提供的命令安装 CodeX。
    *   **付费使用 (ChatGPT Plus)**：需要国外上网环境订阅 ChatGPT Plus，CodeX 启动后登录 ChatGPT 账户即可。
    *   **免费使用 (接入国产模型)**：
        *   在摩大社区等平台注册并获取免费模型的 API 端点和 API Key。
        *   修改 CodeX 配置文件 `config.toml`，填入模型名称、API 端点，并将 API Key 设置为系统环境变量。
        *   **注意**：自定义模型可能存在修改文件时倾向使用 Shell 工具的 Bug，可中断或通过提示词引导解决。
    *   **模型切换**：使用 `/模糊` 命令可在通用模型 (GBT5) 和编程专用模型 (GBT5CodeX) 之间切换，并选择响应速度与推理效果的档位。
    *   **编程案例对比**：以“你画我猜”游戏为例，GPT5 表现最佳（97分），GPT5CodeX 次之（95分），国产模型 JRM4.6 因 Bug 表现一般（80分）。结论是 GPT5 配合 CodeX 的编程效果与 Claude 基本持平。

2.  **ID (集成开发环境插件)**
    *   **安装**：在 VS Code 插件市场搜索并安装官方 CodeX 插件。
    *   **特点**：与命令行环境共享配置和历史记录，提供可视化文件对比、`AWN2` (一键撤销) 功能，并支持在对话中便捷粘贴图片。

3.  **云端环境**
    *   **访问**：通过 CodeX GPT 网页版，绑定 GitHub 账户。
    *   **功能**：可将项目上传至 GitHub，CodeX 会自动在云端创建运行环境，异步执行代码修改任务。
    *   **自动化 Pull Request 与审核**：CodeX 能自动创建 Pull Request 并提交到 GitHub，还提供自动化代码审核功能，帮助提早发现 Bug。

4.  **SDK (软件开发工具包)**
    *   **安装与使用**：初始化 Node.js 项目，安装 `codex-sdk`，编写代码调用 SDK 接口（无需配置 API Key，直接调用本地 CodeX），然后运行 Node.js 脚本。

**三、CodeX 进阶功能与命令**
*   **核心命令**：
    *   `/Init`：通读当前文件，生成 `AZENC.MD` 作为对话上下文，可手动补充项目信息。
    *   `/Agent.MD`：生成更通用的 `AZENC.MD` 文件。
    *   `/Compact`：压缩对话上下文，提高 AI 专注力，降低 Token 消耗。
    *   `/New`：开启新对话，清除历史记录。
    *   `/ApproveMode`：调整运行权限（手动、自动、Full.sx）。
*   **MCP 工具箱 (AI 调用外部工具)**：
    *   通过在 CodeX 配置文件中添加 MCP 服务器配置（如 Context 7, XL MCP Server），AI 可调用外部工具执行任务，如生成 Excel 数据。
*   **自定义命令**：
    *   在 CodeX 配置文件夹的 `Prompts` 目录下，以文件形式创建自定义命令（文件名即命令名），支持添加参数，实现定制化任务（如 Git 分支对比总结）。

**核心观点总结**
CodeX 是一款功能全面、运行环境多样且对中国用户友好的 AI 编程工具，通过集成免费模型和提供自动化协作功能，极大地提升了开发效率和代码质量。
